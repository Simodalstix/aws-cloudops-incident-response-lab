# EC2 CLI Playbook

Focus: launch, inspect, access, control, and basic troubleshooting

---

## launch

```bash
# launch instance
aws ec2 run-instances \
  --image-id <ami-id> \
  --instance-type t2.micro \
  --iam-instance-profile Name=ec2-ssm-profile \
  --security-group-ids <sg-id> \
  --subnet-id <subnet-id> \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=cloudops-lab-01}]'
```

---

## listing

```bash
# list instance IDs
aws ec2 describe-instances \
  --query 'Reservations[].Instances[].InstanceId' \
  --output table

# find by Name tag
aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=cloudops-lab-01"
```

---

## inspection

```bash
# full instance details
aws ec2 describe-instances \
  --instance-ids <instance-id>

# tighter useful view
aws ec2 describe-instances \
  --instance-ids <instance-id> \
  --query 'Reservations[].Instances[].{State:State.Name,PrivateIP:PrivateIpAddress,PublicIP:PublicIpAddress,Type:InstanceType,Profile:IamInstanceProfile.Arn}' \
  --output table

# instance + system health checks
aws ec2 describe-instance-status \
  --instance-ids <instance-id>
```

---

## lifecycle

```bash
# start / stop / reboot / terminate
aws ec2 start-instances \
  --instance-ids <instance-id>

aws ec2 stop-instances \
  --instance-ids <instance-id>

aws ec2 reboot-instances \
  --instance-ids <instance-id>

aws ec2 terminate-instances \
  --instance-ids <instance-id>
```

---

## networking

```bash
# list security groups
aws ec2 describe-security-groups

# inspect one security group
aws ec2 describe-security-groups \
  --group-ids <sg-id>

# check subnet
aws ec2 describe-subnets \
  --subnet-ids <subnet-id>
```

---

## role linkage

```bash
# confirm instance profile attached
aws ec2 describe-instances \
  --instance-ids <instance-id> \
  --query 'Reservations[].Instances[].IamInstanceProfile.Arn' \
  --output table
```

## access / debug

```bash
# boot / console logs
aws ec2 get-console-output \
  --instance-id <instance-id>

# latest screenshot metadata
aws ec2 get-console-screenshot \
  --instance-id <instance-id>

# metadata from inside instance
curl http://169.254.169.254/latest/meta-data/

# useful metadata paths
curl http://169.254.169.254/latest/meta-data/instance-id
curl http://169.254.169.254/latest/meta-data/iam/security-credentials/
```

## quick checks

```bash
# instance exists but app is down:
# -> describe-instance-status
# -> security group rules
# -> ALB target health
# -> SSM access

# cannot reach instance:
# -> running?
# -> correct SG?
# -> public IP?
# -> subnet / route path?
# -> instance profile for SSM?

# weird behaviour:
# -> reboot
# -> console output
# -> CloudWatch metrics
```

## reminders

# running != healthy
# instance status != system status
# security groups are stateful
# IAM / SSM issues often look like app issues
