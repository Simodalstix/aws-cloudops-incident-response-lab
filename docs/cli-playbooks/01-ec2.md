# EC2 CLI Playbook

Focus: launch, inspect, access, control, and basic troubleshooting

---

## pre-launch discovery

```bash
# find latest Amazon Linux 2023 AMI (ap-southeast-2)
aws ec2 describe-images \
  --owners amazon \
  --filters \
    "Name=name,Values=al2023-ami-*-x86_64" \
    "Name=state,Values=available" \
  --query 'sort_by(Images, &CreationDate)[-1].{ID:ImageId,Name:Name,Date:CreationDate}' \
  --output table

# find latest Amazon Linux 2 AMI (legacy reference)
aws ec2 describe-images \
  --owners amazon \
  --filters \
    "Name=name,Values=amzn2-ami-hvm-*-x86_64-gp2" \
    "Name=state,Values=available" \
  --query 'sort_by(Images, &CreationDate)[-1].{ID:ImageId,Name:Name,Date:CreationDate}' \
  --output table

# confirm default VPC exists
aws ec2 describe-vpcs \
  --filters "Name=isDefault,Values=true" \
  --query 'Vpcs[].{VpcId:VpcId,CIDR:CidrBlock,Default:IsDefault}' \
  --output table

# list subnets in default VPC (pick one for launch)
aws ec2 describe-subnets \
  --filters "Name=defaultForAz,Values=true" \
  --query 'Subnets[].{ID:SubnetId,AZ:AvailabilityZone,CIDR:CidrBlock}' \
  --output table

# list security groups (find the right sg-id before launch)
aws ec2 describe-security-groups \
  --query 'SecurityGroups[].{ID:GroupId,Name:GroupName,VPC:VpcId}' \
  --output table

# list key pairs (if SSH fallback is needed)
aws ec2 describe-key-pairs \
  --query 'KeyPairs[].KeyName' \
  --output table

# check t3 instance types available in region
aws ec2 describe-instance-types \
  --filters "Name=instance-type,Values=t3.*" \
  --query 'InstanceTypes[].{Type:InstanceType,CPU:VCpuInfo.DefaultVCpus,RAM:MemoryInfo.SizeInMiB}' \
  --output table
```

> Note: Use AL2023 for new instances. Amazon Linux 2 reaches end of life June 2025.
> Prefer t3.micro over t2.micro — same free tier eligible, better performance.

## launch

```bash
# launch instance (SSM access, no SSH required)
aws ec2 run-instances \
  --image-id <ami-id> \
  --instance-type t3.micro \
  --iam-instance-profile Name=ec2-ssm-profile \
  --security-group-ids <sg-id> \
  --subnet-id <subnet-id> \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=cloudops-lab-01}]'

# launch with key pair (SSH fallback)
aws ec2 run-instances \
  --image-id <ami-id> \
  --instance-type t3.micro \
  --key-name <key-pair-name> \
  --iam-instance-profile Name=ec2-ssm-profile \
  --security-group-ids <sg-id> \
  --subnet-id <subnet-id> \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=cloudops-lab-01}]'
```

## public ip
```bash
# check which subnets have auto-assign public IP enabled
aws ec2 describe-subnets \
  --query 'Subnets[].{ID:SubnetId,AZ:AvailabilityZone,AutoPublicIP:MapPublicIpOnLaunch,CIDR:CidrBlock}' \
  --output table

# force public IP at launch (subnet has auto-assign off)
# note: subnet and SG move into --network-interfaces, not top-level flags
aws ec2 run-instances \
  --image-id <ami-id> \
  --instance-type t3.micro \
  --iam-instance-profile Name=ec2-ssm-profile \
  --network-interfaces "AssociatePublicIpAddress=true,DeviceIndex=0,SubnetId=<subnet-id>,Groups=<sg-id>"

# allocate an Elastic IP (static, survives stop/start)
aws ec2 allocate-address --domain vpc

# attach EIP to running instance
aws ec2 associate-address \
  --instance-id <instance-id> \
  --allocation-id <eip-alloc-id>

# list EIPs
aws ec2 describe-addresses \
  --query 'Addresses[].{IP:PublicIp,AllocID:AllocationId,Instance:InstanceId}' \
  --output table

# release EIP when done (unattached EIPs are billed)
aws ec2 release-address --allocation-id <eip-alloc-id>
```

> Auto-assigned public IPs are ephemeral — they change on stop/start.
> EIPs are static — use them for DNS records, firewall allowlists, anything needing a stable endpoint.
> For SSM-only access you don't need a public IP at all.

## listing

```bash
# list running instances (IDs + names)
aws ec2 describe-instances \
  --filters "Name=instance-state-name,Values=running" \
  --query 'Reservations[].Instances[].{ID:InstanceId,Name:Tags[?Key==`Name`]|[0].Value,State:State.Name}' \
  --output table

# find by Name tag
aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=cloudops-lab-01" \
  --query 'Reservations[].Instances[].InstanceId' \
  --output table

# find by instance profile (confirm SSM-capable instances)
aws ec2 describe-instances \
  --filters "Name=iam-instance-profile.arn,Values=arn:aws:iam::<account-id>:instance-profile/ec2-ssm-profile" \
  --query 'Reservations[].Instances[].{ID:InstanceId,State:State.Name}' \
  --output table
```

## inspection

```bash
# full instance details
aws ec2 describe-instances \
  --instance-ids <instance-id>

# useful summary view
aws ec2 describe-instances \
  --instance-ids <instance-id> \
  --query 'Reservations[].Instances[].{State:State.Name,PrivateIP:PrivateIpAddress,PublicIP:PublicIpAddress,Type:InstanceType,AZ:Placement.AvailabilityZone,Profile:IamInstanceProfile.Arn}' \
  --output table

# instance + system health checks
aws ec2 describe-instance-status \
  --instance-ids <instance-id>

# instance health (both checks in one view)
aws ec2 describe-instance-status \
  --instance-ids <instance-id> \
  --query 'InstanceStatuses[].{Instance:InstanceStatus.Status,System:SystemStatus.Status,State:InstanceState.Name}'
```

## lifecycle

```bash
# start / stop / reboot / terminate
aws ec2 start-instances --instance-ids <instance-id>
aws ec2 stop-instances --instance-ids <instance-id>
aws ec2 reboot-instances --instance-ids <instance-id>
aws ec2 terminate-instances --instance-ids <instance-id>

# wait until running (block until state reached)
aws ec2 wait instance-running --instance-ids <instance-id>
aws ec2 wait instance-stopped --instance-ids <instance-id>
```

## ssm access

```bash
# start interactive session (no SSH, no public IP required)
aws ssm start-session --target <instance-id>

# list instances visible to SSM (confirms agent is registered)
aws ssm describe-instance-information \
  --query 'InstanceInformationList[].{ID:InstanceId,Ping:PingStatus,Platform:PlatformName,AgentVersion:AgentVersion}' \
  --output table

# run a command via SSM (non-interactive)
aws ssm send-command \
  --instance-ids <instance-id> \
  --document-name "AWS-RunShellScript" \
  --parameters 'commands=["uptime","df -h"]'

# get command output
aws ssm get-command-invocation \
  --command-id <command-id> \
  --instance-id <instance-id> \
  --query '{Status:Status,Output:StandardOutputContent}'
```

> If SSM shows no instances: check instance profile is attached, agent is running,
> and outbound 443 is open (or VPC endpoints exist for SSM).

## access / debug

```bash
# boot and console logs
aws ec2 get-console-output \
  --instance-id <instance-id> \
  --latest

# screenshot (useful when instance is unreachable)
aws ec2 get-console-screenshot \
  --instance-id <instance-id>

# metadata from inside the instance
curl http://169.254.169.254/latest/meta-data/
curl http://169.254.169.254/latest/meta-data/instance-id
curl http://169.254.169.254/latest/meta-data/placement/availability-zone
curl http://169.254.169.254/latest/meta-data/iam/security-credentials/

# IMDSv2 (token-based — required on hardened instances)
TOKEN=$(curl -s -X PUT "http://169.254.169.254/latest/api/token" \
  -H "X-aws-ec2-metadata-token-ttl-seconds: 21600")
curl -s -H "X-aws-ec2-metadata-token: $TOKEN" \
  http://169.254.169.254/latest/meta-data/instance-id
```

## quick checks

```bash
# instance exists but app is down:
# → describe-instance-status (both checks)
# → security group rules
# → ALB target health
# → SSM: describe-instance-information

# cannot reach instance:
# → running? (describe-instances)
# → instance profile attached?
# → SSM agent registered? (describe-instance-information)
# → console output for boot errors

# SSM session fails:
# → instance profile missing or wrong role?
# → AmazonSSMManagedInstanceCore attached?
# → outbound 443 open or VPC endpoints configured?
# → agent running? (send-command to check)

# weird behaviour:
# → console output
# → reboot
# → CloudWatch metrics
```
